## 卷积层
全连接神经网络在创建特征向量的时候破坏了图片的空间信息（32 x 32变成了1024 x 1），卷积层就很好的解决了这一问题：
### 组成
**卷积层**的输出是一个3 x 32 x 32的张量（保持image的原有空间结构），这里的3指的是channels(或者叫depth)，比如在图像中这就指的是RGB三种通道
![[Pasted image 20260319100456.png|134]]
**权重矩阵**(或者叫filter)也是一个三维结构（三维小块），其在图片上“滑动”计算*内积*
![[Pasted image 20260319100507.png|409]]
注意一下这里不应该叫矩阵内积，之所以得到**一个数字**，是因为filter和卷积层的对应部分做**Frobenius 内积**（其结果就是矩阵逐元素乘法，然后相加）
卷积在每个通道上分别计算，所以卷积核和输入的图像有**相同的通道数**
![[Pasted image 20260319100433.png]]
### 计算过程
#### 局部
我们在所有的位置（离散的）滑动以后，得到了一个二维的张量(activation map)^[注意，单个卷积会压缩通道数]
![[Pasted image 20260319100712.png|325]]
我们通常会有多个filter（权重不同），下面是六个filter的例子^[**一个卷积核输出（对应）一个Feature Map，多个卷积核对应多个Feature Map**，所以卷积核的个数就是输出Feature Map的通道数]
![[Pasted image 20260319101253.png|495]]
两种视角：
1. 每一个activation map都是image对相应的filter的响应程度
2. activation map也是以六个不同的角度观察image，存储了多个六维的特征向量
实际上，我们是批处理图片的(Batch of images)^[在实际写代码的时候，我们完全不需要在意Batch]
![[Pasted image 20260319101839.png]]
#### 整体
卷积卷积再卷积，实际上不是，卷积的卷积还是卷积$(f*g)*h=f*(g*h)$所以同样的道理，我们需要激活函数（图中$b_1$是6）
![[Pasted image 20260319102324.png]]
输入边长为W，卷积核边长为K，输出的边长为W-K+1

### 问题
注意到我们每次进行卷积操作，都会使得我们的输入边长减少(K-1)，这限制了神经网络的深度(最终退化为一个数)

**解决(Padding-Zero)**：在input的外围添加P圈0，抵消掉卷积带来的维度衰减
![[Pasted image 20260319151317.png|491]]
通常我们设置P使得维度不衰减
### 感受野
![[Pasted image 20260319152419.png]]
感受野（不需要严谨的定义）是随卷积层线性增长的。如图，第四个卷积层内的一块才能有整个输入的视野。如果输入的是一个1024 x 1024的图片，卷积层会非常多（500+）
> [!NOTE] 题外话
> 如果在一个无限区域，你无法观测image是否移动，但是我加了一个边框（边框甚至不可见，这里padding zero就是加边框的人），超过边框的都消失我（变0），这个时候我就能观测到image的移动了。那么卷积在**边界处**就丧失了等变性

## 归一化层
CNN非常深，这使得其难以训练
### Batch Normalize
$$\begin{aligned}

\hat{x}^{(k)}

&=

\frac{x^{(k)}-\mathbb{E}\!\left[x^{(k)}\right]}

{\sqrt{\operatorname{Var}\!\left[x^{(k)}\right]+\varepsilon}}.

\end{aligned}$$

减少了内部协变量偏移，使优化更稳定。这一句话的前半部分是历史上的说法，现在的说法是：1.尺度不敏感（把参数扩大任意倍以后不改变结果），2.梯度（梯度在各个方向都比较接近，偏向圆）
![[Pasted image 20260319170740.png|595]]
实际应用中，我们会在这基础上增加两个可学习的参数（平移 & 缩放），先把所有数据变成“标准坐标系”，然后在这个坐标系里学习最合适的表达
![[Pasted image 20260319181756.png]]
这里有一个问题，我们每一批数据里的数据具体是什么，会影响（因为归一化是受整体的数据影响的）。所以在训练时我们按每一批次来算均值和方差，并记录，在预测的时候使用历史的平均均值和方差作为归一化层的均值和方差（这是一个固定的常数）
由于在**预测时**，均值和方差是常数（对于任意的输入数据都不改变），Batch Normalize是线性的，可以将其融合到前一个线性算子中
![[Pasted image 20260319182828.png]]
经验：使用归一化以后，可以较于不使用归一化的神经网络用更高的学习率而不发散
### 问题
由于归一化的特性，对于期望非常不平衡的输入or输出的数据集，这可能导致问题
不过对于前馈神经网络绝大多数情况都做的很好
